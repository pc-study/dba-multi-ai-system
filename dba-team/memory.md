# 记忆机制配置

DBA团队的记忆机制，用于记住用户的环境配置和历史任务。

## 记忆内容

### 1. 环境配置 (env)
- 用户有哪些数据库
- 各数据库的版本
- 集群架构信息
- 连接信息（脱敏后）

### 2. 历史任务 (history)
- 过去处理过的任务
- 执行的方案
- 任务结果

### 3. 用户偏好 (preferences)
- 偏好的数据库
- 常用的配置
- 关注的问题类型

## 存储位置

```
~/.openclaw/workspace/dba-multi-ai-system/memory/
├── env.json           # 环境配置
├── history.json      # 历史任务
└── preferences.json  # 用户偏好
```

## 自动记忆规则

1. **环境信息**：首次提到数据库环境时记录
2. **任务结果**：每次任务完成后记录
3. **用户偏好**：通过对话学习

## 记忆调用

在SKILL.md中可通过读取memory目录获取上下文：
- 了解用户有哪些数据库
- 参考之前的处理方案
- 记住用户的偏好设置
